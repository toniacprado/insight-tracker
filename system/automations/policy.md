# Automation Policy
*Version:* v0.4  
*Date:* 2026-03-11  
*Last reviewed:* 2026-03-11

State what automation is allowed to do in this repo.

Default policy:
- draft-first, not send-first
- reviewable outputs
- explicit logging for automated writes
- respect data sensitivity boundaries from `docs/DATA_POLICY.md`
- respect the guardrails in `docs/GUARDRAILS.md`
- no destructive or externally visible action without explicit approval
