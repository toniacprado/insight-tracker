*Version:* v0.2  
*Date:* 2026-03-11  
*Last reviewed:* 2026-03-11

## Terms
- Canonical: the source-of-truth version that future work should trust.
- Contract drift: when code, docs, prompts, or policies no longer describe the same
  behavior.
- Eval case: a scoped example used to verify prompt or agent behavior.
- Prompt asset: a versioned runtime prompt stored in the repo.
- Runtime artifact: generated output that can be rebuilt and should not be canonical.
- System asset: a reusable schema, template, prompt, or policy under `system/`.
